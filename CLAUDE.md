# Sinhala Food Ordering AI — Project Guide for Claude

## Project Overview
Advanced AI university group project. A Sinhala-speaking conversational AI agent that handles food orders through **WhatsApp** and **phone calls**. Deadline: 2026-06-22.

**Shop:** Wakwalle Kade (වක්වැල්ලේ කඩේ) — food shop near university campus, Hapugala, Wakwalle, Galle.
**No delivery — pickup only. No spice levels on any item.**

## Tech Stack
- **LLM:** Gemma 4 E2B (`google/gemma-4-e2b-it`) fine-tuned with LoRA via Unsloth
- **Fine-tuning platform:** Lightning AI / VPS with GPU (geo-blocked in Sri Lanka — use VPN or VPS)
- **Voice:** Twilio (STT via `<Gather language="si-LK">`) + Google Cloud TTS (`si-LK-Standard-A`)
- **WhatsApp:** Twilio sandbox
- **RAG:** FAISS + `sentence-transformers` (`all-MiniLM-L6-v2`) — hand-rolled, no LangChain
- **Backend:** FastAPI + uvicorn
- **Webhook tunnel:** ngrok (local dev)
- **Order logging:** CSV (`data/orders.csv`)

## Menu Structure (menu_data.json)
Each item uses ONE of three pricing structures — no spice levels anywhere:

| Item | Structure | Example |
|------|-----------|---------|
| Rice & Curry | `options[]` — pick protein | veg Rs.250 / egg Rs.290 / omelette Rs.320 / fish Rs.350 / chicken Rs.400 |
| String Hoppers | `packages[]` — pick count | 10pcs Rs.150 / 15pcs Rs.225 / 20pcs Rs.300 / 30pcs Rs.450 |
| Kottu (chicken/egg/fish) | `portions[]` — half or full | Chicken: half Rs.450 / full Rs.800 |
| Fried Rice (chicken/egg/fish) | `portions[]` — half or full | Chicken: half Rs.450 / full Rs.800 |

## Ordering Flow
1. Customer states item
2. Agent asks the right clarifying question:
   - Rice & Curry → **which protein?** (veg/egg/omelette/fish/chicken)
   - Kottu / Fried Rice → **half or full?**
   - String Hoppers → **which package?** (10/15/20/30)
3. Agent asks quantity
4. Agent collects **phone number** (no address — pickup only)
5. Agent shows full order summary with total
6. Customer confirms → order saved to `data/orders.csv` with order ID `#WK00001`

## Voice Call Flow
```
Customer calls → Twilio → POST /voice/incoming
  → TwiML: <Gather language="si-LK"> plays greeting MP3
  → Caller speaks Sinhala/English
  → Twilio STT transcribes → POST /voice/process (SpeechResult=...)
  → agent.respond() → Gemma 4 + RAG → reply text
  → Google TTS → MP3 → <Play> URL
  → Loop: listen again
```

## WhatsApp Flow
```
Customer texts → Twilio → POST /whatsapp/incoming (Body=..., From=whatsapp:+94...)
  → agent.respond() → Gemma 4 + RAG → reply text
  → Twilio MessagingResponse → sends WhatsApp reply
```

## RAG Architecture (hand-rolled, no LangChain)
```
Startup: menu_data.json → 8 item text strings → SentenceTransformer encode → FAISS IndexFlatL2
Query:   user_message → encode → FAISS search top-4 → format as text → injected into system prompt
```
- Model: `all-MiniLM-L6-v2` (384-dim embeddings)
- Each item text includes name, Sinhala name, category, description, and full pricing
- RAG context injected as `{menu_context}` in SYSTEM_PROMPT on every turn

## Dataset
- 100 synthetic Sinhala conversations in `data/synthetic_dataset.json` (conv_001–conv_100)
- 4 language styles: formal_sinhala / informal_sinhala / sinhala_english_mixed / english_with_sinhala
- No spice levels. Rice & Curry always asks protein. Phone number always collected. Pickup only.
- Training format: ShareGPT (`conversations: [{role, content}]`)
- Intents: food_order, menu_inquiry, price_inquiry, portion_inquiry, protein_select, package_select, phone_provide, confirm_order, cancel_order, quantity_change, availability_inquiry

## How to Run

### Terminal demo (fastest — no credentials needed)
```bash
pip install -r requirements.txt
python app/demo_terminal.py
```

### Full system (needs .env filled)
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
ngrok http 8000   # in another terminal
# paste ngrok URL into Twilio sandbox webhook settings
```

### Fine-tuning (VPS with GPU / Google Colab)
```bash
# On VPS or Colab with L4/T4 GPU:
pip install unsloth trl peft accelerate bitsandbytes datasets huggingface_hub evaluate rouge-score matplotlib
# Upload data/ folder, then run notebook:
jupyter notebook notebooks/02_fine_tuning_gemma4.ipynb
# OR convert to script and run:
jupyter nbconvert --to script notebooks/02_fine_tuning_gemma4.ipynb
python notebooks/02_fine_tuning_gemma4.py
```
After training: copy HuggingFace repo ID → `HF_MODEL_ID` in `.env`

## Important Notes
- Do NOT use Anthropic API — use the fine-tuned Gemma 4 model only
- Lightning AI is geo-blocked in Sri Lanka — use a VPS or Google Colab for fine-tuning
- Google TTS voice for Sinhala: `si-LK-Standard-A`
- Twilio `<Gather language="si-LK">` handles Sinhala STT automatically (no direct Google STT needed)
- Session state is in-memory (keyed by phone number), expires after 30 min
- Audio files are cached by MD5 hash of text — won't re-generate same phrase
- No minimum order, no delivery fee — pickup only
- `data/orders.csv` is auto-created on first confirmed order
